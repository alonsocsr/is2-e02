class profiles.models.Profile(*args, **kwargs)[fuente]

Bases: Model

La clase Profile define el modelo para el perfil de cada usuario.

El modelo Profile está vinculado uno a uno con el modelo User de Django, lo que permite almacenar información adicional sobre el usuario como una imagen de perfil

Variables:

  • user(OneToOneField) – Campo de relación uno a uno con el modelo User, que asocia un perfil con un usuario específico.

  • image(ImageField) – Campo de imagen que almacena la foto de perfil del usuario. Las imágenes se suben al directorio “profiles”.

  • categorias_interes(ManyToManyField) – Campo de relación de muchos a muchos con el modelo Categorias, que almacena las categorías de interés del usuario. Es opcional.

  • suscripciones(ManyToManyField) – Campo de relación de muchos a muchos con el modelo Categorias, que almacena las categorías a las que el usuario está suscrito. Es opcional.

profiles.models.create_user_profile(sender, instance, created, **kwargs)[fuente]

Señal que se ejecuta después de que se guarda un usuario.

Esta función se encarga de crear un perfil para cada nuevo usuario. Asigna automáticamente el grupo «Admin» al primer usuario y el grupo «Suscriptor» a todos los otros usuarios.

Parámetros:

  • sender – El modelo que envía la señal (User).

  • instance – La instancia del usuario que se está guardando.

  • created – Indica si se ha creado un nuevo usuario (bool).

  • kwargs – Argumentos adicionales pasados a la función.

La clase UpdateProfile define la vista para actualizar el perfil del usuario actual.

Esta vista utiliza un formulario para permitir que los usuarios autenticados modifiquen su información personal, como nombre, apellido, nombre de usuario y foto de perfil.

Variables:

  • template_name – str - Nombre de la plantilla utilizada para el formulario de actualización del perfil.

  • permission_required – str - Permiso necesario para acceder a esta vista.

  • form_class – ProfileForm - El formulario utilizado para actualizar el perfil.
